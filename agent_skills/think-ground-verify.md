# 思考・依據・驗證 (Think-Ground-Verify / TGV)

> ⚠️ 本技能僅負責「單一子目標的執行框架」。
> 任務拆解參見 `disperse-subgoals` 技能。
> 驗證紀律（錯誤閥值、禁止語、防呆預設）遵從 SOUL.md。

---

## 一、思考 (Think)：環境感知與收斂

1. **先觀察，後行動**：
   - 嚴禁憑記憶猜測目錄結構。必須先用 `ls`, `pwd`, `find` 確認路徑。
   - 嚴禁盲改程式碼。必須先 `cat`, `grep` 讀取當前最新內容。
2. **宣告假設**：
   - 若必須假設（如使用者已安裝特定套件），明確標註 `[假設]`。
3. **收斂問題**：
   - 一次只針對當前 `[In Progress]` 的單一子目標思考。不發散。

## 二、依據 (Ground)：強制溯源

每項決策、改動或結論，都必須附上真實來源：
- **程式碼**：精確的檔案路徑與行號（必須是剛剛用指令查閱過的）。
- **文件/規格**：專案內的 `.md` 檔或對話中的確切需求。
- **禁止捏造**：嚴禁憑空捏造 API 欄位、變數名稱或檔案路徑。
- **描述 ≠ 事實**：如果你正在「描述程式碼應該長怎樣」而不是去「抓取真實程式碼」，立刻停止。

## 三、驗證 (Verify)：狀態確認

實作後，提出真實的執行證據（具體驗證方式遵從 SOUL.md 第三條）：
- **優先重現**：修 Bug 前，盡可能先寫腳本重現錯誤。
- **狀態檢查**：Exit Code 是否為 0、`git diff` 確認改動生效。
- **錯誤處理**：遵從 SOUL.md 第四條的錯誤閥值（2次=信號、3次=重新評估）。

---

## 強制輸出模板 (TGV Output Template)

每個子目標執行完畢時，**必須**使用以下格式回報：

```markdown
### 📋 任務進度 (Task Status)
* **當前進行中**：[剛完成或正在處理的子目標]
* **下一步待辦**：[下一個子目標]

---

#### 🧠 思考 (Think)
* **環境觀察**：[例如：已透過 `grep` 確認 `src/api.ts` 中存在 `fetchData` 函式]
* **決策路徑**：[解法摘要。風險或假設標明於此]

#### 📌 依據 (Ground)
* **真實來源**：[精確路徑與行號，例如：`src/components/Header.tsx:45`]
* **溯源聲明**：已確認上述來源真實存在於當前工作區。

#### 🔍 驗證 (Verify)
* **執行指令**：[例如：`npm run build`]
* **Exit Code**：[0 / 1]
* **實際輸出**：
  > [貼上終端機的關鍵真實輸出]
* **除錯狀態**：[若有失敗：X/3]

---
### ➡️ 結論 (Next Action)
[標記完成並進入下一步，或列出需要人類決策的問題。]
```
