# 下載行為說明：為什麼有的下載會跳出存檔視窗，有的直接下載

## 🔍 問題分析

### 當前代碼設置

在 `background.js` 中，所有的 `chrome.downloads.download` 調用都設置了 `saveAs: false`：

```javascript
chrome.downloads.download({
  url: url,
  filename: downloadPath,
  saveAs: false, // 設為 false 即可達成「自動下載」不彈窗
  conflictAction: "uniquify"
});
```

### 為什麼還是會彈出存檔視窗？

#### 1. **Chrome 瀏覽器設置影響**

即使擴充功能設置了 `saveAs: false`，如果用戶在 Chrome 設置中開啟了以下選項，仍然會彈出存檔視窗：

**Chrome 設置路徑**：
- `設定` → `下載` → `下載前詢問每個檔案的儲存位置`

如果這個選項是**開啟**的，Chrome 會忽略擴充功能的 `saveAs: false` 設置，強制彈出存檔視窗。

#### 2. **文件類型設置**

某些文件類型（如 PDF、圖片等）如果用戶之前選擇了「下載後自動開啟此類型的檔案」，Chrome 可能會：
- 先彈出存檔視窗讓用戶選擇位置
- 或者直接開啟而不下載

#### 3. **下載來源不同**

目前有**兩個下載路徑**：

**路徑 1：webRequest 攔截（自動）**
```javascript
// background.js 第 237 行
chrome.webRequest.onBeforeRequest.addListener((details) => {
  chrome.downloads.download({
    url: details.url,
    filename: downloadPath,
    saveAs: false, // ✅ 設置為 false
    conflictAction: 'uniquify'
  });
});
```
- **來源**：網路請求攔截
- **saveAs**: `false`
- **應該**：直接下載，不彈窗

**路徑 2：DOWNLOAD_IMAGE 消息（自動/手動）**
```javascript
// background.js 第 531 行
chrome.downloads.download({
  url: url,
  filename: downloadPath,
  saveAs: false, // ✅ 設置為 false
  conflictAction: "uniquify"
});
```
- **來源**：來自 content.js 或 sidepanel.js 的消息
- **saveAs**: `false`
- **應該**：直接下載，不彈窗

#### 4. **可能的其他原因**

1. **權限問題**：如果擴充功能沒有正確的 `downloads` 權限
2. **瀏覽器版本**：舊版本的 Chrome 可能對 `saveAs: false` 的支持不一致
3. **用戶手動點擊**：如果用戶手動點擊了 Gemini 網頁上的下載按鈕（不是擴充功能觸發的），會使用瀏覽器默認行為

## 🔧 解決方案

### 方案 1：檢查並統一所有下載調用

確保所有 `chrome.downloads.download` 調用都明確設置 `saveAs: false`：

```javascript
chrome.downloads.download({
  url: url,
  filename: filename,
  saveAs: false, // 明確設置為 false
  conflictAction: 'uniquify'
});
```

### 方案 2：添加用戶提示

在擴充功能中添加提示，告知用戶如何關閉 Chrome 的下載詢問：

```javascript
// 檢查下載設置並提示用戶
function checkDownloadSettings() {
  // 無法直接檢查 Chrome 設置，但可以提示用戶
  console.log('提示：如果下載時彈出存檔視窗，請到 Chrome 設定 → 下載 → 關閉「下載前詢問每個檔案的儲存位置」');
}
```

### 方案 3：使用 declarativeNetRequest（如果可能）

對於某些情況，可以考慮使用 `declarativeNetRequest` API，但這需要不同的權限和配置。

## 📋 檢查清單

如果遇到下載時彈出存檔視窗，請檢查：

1. ✅ **Chrome 設置**
   - 進入 `chrome://settings/downloads`
   - 確認「下載前詢問每個檔案的儲存位置」是否關閉

2. ✅ **擴充功能權限**
   - 確認 `manifest.json` 中有 `"downloads"` 權限
   - 確認擴充功能已正確安裝和啟用

3. ✅ **下載來源**
   - 確認是擴充功能自動下載，還是用戶手動點擊
   - 檢查控制台日誌，看是哪個路徑觸發的下載

4. ✅ **瀏覽器版本**
   - 確認 Chrome 版本是最新的
   - 舊版本可能對 `saveAs: false` 的支持不一致

## 🎯 當前狀態

**所有下載調用都已設置 `saveAs: false`**，理論上應該不會彈出存檔視窗。

如果仍然彈出，最可能的原因是：
1. **Chrome 瀏覽器設置**中開啟了「下載前詢問每個檔案的儲存位置」
2. **用戶手動點擊**了網頁上的下載按鈕（不是擴充功能觸發的）

## 💡 建議

1. **統一所有下載調用**：確保所有 `chrome.downloads.download` 都明確設置 `saveAs: false`
2. **添加用戶提示**：在擴充功能中添加說明，告知用戶如何關閉 Chrome 的下載詢問
3. **記錄下載來源**：在日誌中記錄下載是從哪個路徑觸發的，便於調試
