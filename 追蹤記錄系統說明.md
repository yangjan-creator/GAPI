# 完整追蹤記錄系統說明

## 📋 系統概述

這套追蹤記錄系統專門設計用於詳細記錄下載按鈕的點擊過程，收集所有關鍵信息，以便後續分析和優化圖片下載按鈕的設計。

## 🎯 設計目標

1. **完整記錄**：記錄按鈕點擊的完整過程，包括 DOM 結構、網絡請求、重定向鏈等
2. **結構化數據**：使用 JSON 格式，便於後續分析和處理
3. **易於導出**：提供一鍵導出功能，方便提供給 AI 分析
4. **詳細信息**：包含足夠的細節，以便理解按鈕行為和下載流程

## 📊 記錄內容

### 1. 按鈕基本信息

當按鈕被點擊時，系統會記錄：

- **基本屬性**：
  - `tagName`: 標籤名
  - `id`: 元素 ID
  - `className`: 類名
  - `classList`: 類名數組
  
- **屬性信息**：
  - `attributes`: 所有屬性
  - `dataAttributes`: 所有 data-* 屬性
  - `ariaAttributes`: 所有 aria-* 屬性
  
- **內容信息**：
  - `innerHTML`: 內部 HTML（前 500 字符）
  - `innerText`: 文本內容（前 200 字符）
  - `outerHTML`: 完整 HTML（前 1000 字符）
  
- **位置和尺寸**：
  - `boundingRect`: 元素在頁面中的位置和尺寸
    - `x`, `y`, `width`, `height`
    - `top`, `left`, `right`, `bottom`
  
- **樣式信息**：
  - `computedStyle`: 計算後的樣式
    - `display`, `visibility`, `opacity`
    - `position`, `zIndex`
    - `cursor`, `pointerEvents`
    - `backgroundColor`, `color`
    - `fontSize`, `fontWeight`
    - `padding`, `margin`, `border`
  
- **DOM 結構**：
  - `parentElement`: 父元素信息
  - `siblings`: 兄弟元素信息
  - `children`: 子元素信息（前 5 個）
  
- **特殊屬性**：
  - `jslog`: jslog 屬性值
  - `dataTestId`: data-test-id 屬性值
  - `ariaLabel`: aria-label 屬性值
  - `role`: role 屬性值
  - `disabled`: 是否禁用
  - `hidden`: 是否隱藏

### 2. 點擊事件信息

- `clickCoordinates`: 點擊座標
  - `x`, `y`: 相對於視口的座標
  - `pageX`, `pageY`: 相對於頁面的座標
- `clickTimestamp`: 點擊時間戳
- `isUserClick`: 是否為用戶真實點擊（vs 程序觸發）
- `isTestClick`: 是否為測試點擊

### 3. 網絡請求信息

每次網絡請求都會記錄：

- **請求信息**：
  - `url`: URL（前 500 字符）
  - `fullUrl`: 完整 URL（用於分析）
  - `method`: HTTP 方法
  - `headers`: 請求頭鍵名列表
  
- **URL 模式分析**：
  - `urlPattern`: URL 結構化信息
    - `protocol`: 協議
    - `hostname`: 主機名
    - `pathname`: 路徑（前 200 字符）
    - `searchParams`: 查詢參數（前 10 個）
    - `hasGgDl`: 是否包含 gg-dl
    - `hasRdGgDl`: 是否包含 rd-gg-dl
    - `hasS0`: 是否包含 =s0 或 s0-d-I
    - `hasAlr`: 是否包含 alr=yes
    - `urlLength`: URL 長度
  
- **時間信息**：
  - `timeSinceClick`: 距離點擊的時間（毫秒）
  - `step`: 重定向步驟

### 4. URL 重定向鏈

完整記錄 URL 重定向過程：

- **重定向信息**：
  - `fromUrl`: 來源 URL（前 500 字符）
  - `fullFromUrl`: 完整來源 URL
  - `toUrl`: 目標 URL（前 500 字符）
  - `fullToUrl`: 完整目標 URL
  - `fromUrlPattern`: 來源 URL 模式
  - `toUrlPattern`: 目標 URL 模式
  - `responseStatus`: HTTP 狀態碼
  - `responseStatusText`: HTTP 狀態文本
  - `responseHeaders`: 響應頭

### 5. DOM 變化

記錄按鈕及其周圍元素的變化：

- `type`: 變化類型（attributes, childList, characterData）
- `target`: 目標元素標籤名
- `targetClass`: 目標元素類名
- `attributeName`: 變化的屬性名
- `oldValue`: 舊值
- `addedNodes`: 添加的節點數量
- `removedNodes`: 移除的節點數量

### 6. 按鈕狀態變化

記錄按鈕狀態的變化：

- `className`: 類名
- `ariaLabel`: aria-label
- `hasSpinner`: 是否有加載動畫
- `disabled`: 是否禁用
- `timeSinceClick`: 距離點擊的時間

### 7. 頁面環境信息

每個記錄都包含：

- `chatId`: 對話 ID
- `userProfile`: 用戶檔案
- `url`: 頁面 URL（前 200 字符）
- `pageTitle`: 頁面標題（前 200 字符）
- `userAgent`: 用戶代理（前 200 字符）
- `viewport`: 視口尺寸
  - `width`: 寬度
  - `height`: 高度

## 🚀 使用方法

### 1. 開始記錄

系統會自動記錄所有下載按鈕的點擊事件，無需手動啟動。

### 2. 查看記錄

在側邊欄的「📹 點擊監聽記錄」區域可以查看所有記錄。

### 3. 導出記錄

點擊「💾 導出記錄」按鈕，系統會：
1. 收集所有記錄
2. 生成 JSON 文件
3. 自動下載到瀏覽器的默認下載目錄

文件名格式：`gemini_click_monitor_{userProfile}_{timestamp}.json`

### 4. 提供給 AI 分析

將導出的 JSON 文件提供給 AI，AI 可以：
1. 分析按鈕的 DOM 結構
2. 理解點擊流程
3. 識別 URL 重定向模式
4. 設計更優化的下載按鈕邏輯

## 📝 事件類型

系統記錄以下事件類型：

- `BUTTON_CLICKED`: 按鈕被點擊
- `TEST_BUTTON_CLICKED`: 測試點擊
- `BUTTON_STATE_INITIAL`: 按鈕初始狀態
- `BUTTON_STATE_CHANGED`: 按鈕狀態變化
- `NETWORK_REQUEST_FETCH`: Fetch 網絡請求
- `NETWORK_REQUEST_XHR`: XHR 網絡請求
- `URL_REDIRECT`: URL 重定向
- `URL_FOUND_IN_RESPONSE`: 在響應中找到 URL
- `DOWNLOAD_TEXT_DETECTED`: 檢測到下載文本
- `DOWNLOAD_STARTED`: 下載開始
- `TRACK_STEP`: 追蹤步驟
- `TRACK_REDIRECT`: 追蹤重定向
- `TRACK_URL_EXTRACTED`: 追蹤 URL 提取
- `TRACK_FINAL_URL`: 追蹤最終 URL
- `TRACK_MAX_STEPS_REACHED`: 達到最大追蹤次數
- `TRACK_ERROR`: 追蹤錯誤
- `MONITOR_ENDED`: 監聽結束

## 🔍 分析建議

當您提供記錄文件給 AI 時，AI 可以：

1. **分析按鈕結構**：
   - 識別按鈕的共同特徵
   - 找到最可靠的選擇器
   - 理解按鈕的 DOM 結構

2. **理解下載流程**：
   - 分析 URL 重定向模式
   - 識別關鍵的 URL 參數
   - 找到最終下載 URL 的規律

3. **優化下載邏輯**：
   - 設計更高效的按鈕檢測方法
   - 優化 URL 提取邏輯
   - 改進重定向追蹤策略

4. **識別問題**：
   - 找出失敗的原因
   - 識別邊緣情況
   - 優化錯誤處理

## ⚙️ 技術細節

### 批量發送優化

- 使用防抖機制，每 500ms 批量發送一次記錄
- 當待發送記錄達到 20 條時立即發送
- 減少消息傳遞次數，提高性能

### URL 去重

- 使用 Set 記錄正在追蹤的 URL
- 避免重複追蹤同一個 URL
- 30 秒後自動清理，防止永久佔用

### 數據大小限制

- 字符串超過 1000 字符會被截斷
- URL 長度限制為 200 字符（顯示用）
- 完整 URL 保存在 `fullUrl` 字段中

## 📌 注意事項

1. **記錄數量限制**：最多保留 100 條記錄（內存中）和 200 條記錄（持久化）
2. **性能影響**：記錄系統經過優化，對性能影響最小
3. **隱私保護**：記錄中包含頁面 URL 和標題，請注意隱私

## 🎉 總結

這套追蹤記錄系統提供了完整的按鈕點擊和下載流程記錄，包含所有必要的細節信息。導出的 JSON 文件可以直接提供給 AI 進行分析，幫助設計更優化的圖片下載按鈕。
